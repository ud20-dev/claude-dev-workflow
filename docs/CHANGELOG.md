# CHANGELOG
> Journal humain du projet — écrit pour qu'un développeur qui découvre le projet comprenne son histoire et son état en le lisant, phrase par phrase.
> Différent de PROGRESS.md : PROGRESS.md est la mémoire de travail de Claude (terse, pointeurs, lu au début de chaque session) — ce fichier est pour un humain, en langage clair, jamais lu automatiquement par Claude sauf demande explicite.
> Une entrée par changement significatif (nouvelle fonctionnalité, décision qui change la direction, gros refactor) — pas pour chaque petit commit.
> La plus récente en haut. Ne jamais supprimer une entrée existante.
> Format d'ajout : "Ajoute dans CHANGELOG.md — [ce qui a changé, en une phrase]"

---

## Aperçu rapide
> Vue d'ensemble en un coup d'œil — mise à jour à chaque entrée de journal ajoutée en dessous.

| Zone | État actuel | Dernière mise à jour |
|------|-------------|----------------------|
| Frontend | | JJ/MM/AAAA |
| Backend | | JJ/MM/AAAA |
| Base de données | | JJ/MM/AAAA |
| Déploiement | | JJ/MM/AAAA |

---

## Template
```
### JJ/MM/AAAA — [Titre court, compréhensible sans contexte]
[2 à 4 phrases complètes : qu'est-ce qui a changé, pourquoi, qu'est-ce que ça permet maintenant. Écrire pour quelqu'un qui n'a pas suivi le projet — pas de jargon interne non expliqué, pas de pointeur du style "voir X.md" sans dire ce qu'il y a dedans.]
```
